# kd-dialogues

A tiny YAML-driven engine that turns a written dialogue between political thinkers into three synchronised videos (16:9 / 9:16 / 1:1) with mixed voice tracks, then packages them for GitHub, Substack, YouTube, TikTok, Instagram, and X.

Built from the *SIX THINKERS: DIALOGUES* promo for [The Kiwi Dialectic](https://kiwidialectic.substack.com). Extensible: drop a new figure into `thinkers/`, write a new `script.yml`, run one command.

---

## What this repo is

- **`kd/`** — the engine (Python)
  - `generate.py` — the video pipeline (three-aspect-ratio pencil-sketch dialogues)
  - `render_16x9.py`, `render_9x16.py`, `render_1x1.py` — one video renderer per aspect ratio
  - `render_common.py` — shared paper/portrait/caption primitives
  - `tts.py` — voice synthesis wrapper (sandbox `asi-text-to-speech`, `KD_TTS_CMD`, or silent placeholder)
  - `publish.py` — video distribution: GitHub, Substack, per-platform export pack, YouTube
  - **`audio/`** — the audio-only pipeline for long-form MP3 episodes:
    - `render.py` — TTS every cue, stitch to one MP3 with ID3 tags, chapter markers, and a Substack embed
    - `waveform.py` — cream-paper-and-ink waveform PNG for the site/embed
    - `draft.py` — skeleton generator: topic + cast → `script.yml` with each thinker's vernacular hints as comments
  - `publish_audio.py` — audio distribution: GitHub push, Substack embed with real raw URL baked in, export-pack with captions and chapter show notes
- **`thinkers/`** — the extensible library. Each subdirectory holds one figure:
  ```
  thinkers/gramsci/
    portrait.png
    voice.yml         # display_name, tts_voice, cadence, vernacular hints, sources
  ```
- **`dialogues/`** — video content. One directory per dialogue; the only thing an author edits is `script.yml`:
  ```
  dialogues/001-first-round/
    script.yml        # ← you edit this
    audio/            # generated (per-line mp3s)
    video/            # generated (three mp4s + audio_mix + metadata + embed)
    dist/             # generated by `publish.py` (captions + manifest + checklist)
  ```
- **`episodes/`** — audio content. Long-form multi-thinker MP3 dialogues:
  ```
  episodes/002-india-fta/
    script.yml                    # ← you edit this (see episodes/SCHEMA.md)
    audio/                        # generated (per-cue mp3s)
    india-fta.mp3                 # stitched episode
    india-fta.waveform.png        # cream-paper waveform
    transcript.md                 # generated
    metadata.json                 # runtime + chapter markers
    embed.html                    # site embed (uses {MP3_URL} placeholder)
    embed.substack.html           # generated by publish_audio (URL baked in)
    dist/                         # captions + upload checklist
  ```
- **`docs/`** — a static GitHub Pages catalogue index of every dialogue. Rebuild with `python docs/build_site.py`.

## Audio-only quickstart (long-form MP3 episodes)

Use this when you want two-to-six thinkers to talk at length (5–30+ min) with no visuals — just voice, one waveform PNG, and a Substack-embeddable player.

```bash
# 1. Draft a script skeleton from a topic + cast
python -m kd.audio.draft \
    --slug india-fta \
    --title "A once-in-a-generation opportunity?" \
    --cast luxon,freire \
    --acts "I. Diagnosis,II. Disagreement,III. Proposal" \
    --turns-per-act 2

# 2. Edit episodes/NNN-<slug>/script.yml — replace the prompts with real lines.
#    Each thinker's `vernacular:` and `delivery_tags:` from their voice.yml are
#    inlined as YAML comments, so you write in-voice.

# 3. Render — TTS every cue, stitch to MP3, generate waveform + transcript
python -m kd.audio.render episodes/NNN-<slug>/script.yml --transcript

# 4. Publish
python -m kd.publish_audio episodes/NNN-<slug> --target all
```

**Outputs**: one stitched MP3 with full ID3 tags (title/album/artist/genre/license+cast), a 1600×900 cream-paper waveform PNG in the site palette, a `transcript.md`, a `metadata.json` with chapter markers, and an `embed.html` you can paste straight into Substack (via `--target substack` the `{MP3_URL}` placeholder is filled with the real GitHub raw URL).

**Schema**: see `episodes/SCHEMA.md`. A script has a `cast` (2–6 thinker slugs), optional `narrator_voice`, `cold_open`, `end_card`, and a list of `acts`, each with an `intro` and a list of `turns` (`thinker` + `say` + optional `tail_ms` beat).

**Adding a new thinker**: drop `thinkers/<slug>/portrait.png` + `voice.yml` and you can immediately cast them in an episode. See `thinkers/luxon/voice.yml` for a full example including `positions`, `signature_lines`, and `rhetorical_moves` so a script author (or an LLM) can argue against the real position instead of a strawman.

## Author quickstart (video pipeline)

### 1. Create a dialogue

```bash
mkdir -p dialogues/002-my-topic
cat > dialogues/002-my-topic/script.yml <<'YAML'
slug: my-topic
title: SIX THINKERS: DIALOGUES
subtitle: A writing-room course from The Kiwi Dialectic.
cold_open:  ROUND ONE
turn_card:  THE TURN
end_card:
  line_1: One writing room. Six figures.
  url_1:  kiwidialectic.substack.com
  url_2:  github.com/robertmccallnz/kd-dialogues
lines:
  - thinker: gramsci
    say: |
      [deliberate] The problem is not the state. [pauses] The problem is what the state
      has already taught us to want.
    caption: The problem is what the state has already taught us to want.
  - thinker: kropotkin
    say: |
      [warm] Every village that repaired its own road [pauses] was already practising
      the future you're describing.
    caption: Every village that repaired its own road was already practising the future.
  # … add more lines. Any thinker with a folder under `thinkers/` is fair game.
YAML
```

### 2. Generate

```bash
python -m kd.generate dialogues/002-my-topic/script.yml
```

That runs, in order:

1. Synthesise each line to `audio/<slug>.mp3` (using the voice bound to that thinker in `thinkers/<slug>/voice.yml`).
2. Mix all lines into `video/audio_mix.m4a` with beat-accurate delays.
3. Render three mp4s (16:9 / 9:16 / 1:1) into `video/`.
4. Emit `video/metadata.json` and `video/embed.html`.

**Long renders** — each aspect ratio takes ~5–8 minutes at 24 fps. If you're on a shell with a strict wall-clock limit, render one at a time:

```bash
python -m kd.generate dialogues/002-my-topic/script.yml --aspects 16x9
python -m kd.generate dialogues/002-my-topic/script.yml --skip-tts --aspects 9x16
python -m kd.generate dialogues/002-my-topic/script.yml --skip-tts --aspects 1x1
```

`--skip-tts` reuses the mp3s already sitting in `audio/`. The sidecar files (`metadata.json`, `embed.html`) auto-merge whatever mp4s exist on disk.

### 3. Publish

```bash
python -m kd.publish dialogues/002-my-topic --target export-pack   # captions + manifest
python -m kd.publish dialogues/002-my-topic --target substack      # bakes GitHub raw URL into embed
python -m kd.publish dialogues/002-my-topic --target github        # commit + push
python -m kd.publish dialogues/002-my-topic --target youtube       # (needs YouTube connector)
python -m kd.publish dialogues/002-my-topic --target all           # all of the above
```

The **export-pack** target writes:

```
dialogues/002-my-topic/dist/
  captions/{youtube,tiktok,instagram,x}.txt
  manifest.json
  upload-checklist.md
```

`upload-checklist.md` tells you exactly which mp4 + caption to hand-post on each platform where no API upload is available (TikTok, Instagram, X on free tier).

## Adding a new thinker

Everything about a voice lives in one folder:

```
thinkers/luxemburg/
  portrait.png              # 1:1, warm cream-paper illustration, ~1024×1024
  voice.yml
```

`voice.yml`:

```yaml
display_name: Rosa Luxemburg
tts_voice: alnilam            # any Gemini TTS voice; see below
cadence: forceful, staccato
vernacular_hints:
  - working-class German syntax
  - "not …, but …" antitheses
delivery_tags_preferred:
  - "[emphatic]"
  - "[pauses]"
replacement_phrasings:
  - reformism → "the illusion of small steps"
sources:
  - "Reform or Revolution (1900)"
```

Then reference `thinker: luxemburg` in any `script.yml`. That's it.

**Portraits** — the renderers assume a warm hand-drawn pencil-sketch look on cream paper. Any square image works, but for aesthetic consistency: soft graphite tones, mid-value background, no busy details.

**Voices** — see `kd/tts.py` for the backend contract. Inside a Perplexity sandbox the voice string maps to Gemini TTS names (`iapetus`, `charon`, `alnilam`, `sadaltager`, `enceladus`, `puck`, `kore`, `orus`, `zubenelgenubi`, `algieba`, `algenib`, `laomedeia`, `achernar`, `schedar`, `gacrux`, `pulcherrima`, `umbriel`, `callirrhoe`, `autonoe`, `despina`, `erinome`, `rasalgethi`, `sulafat`, `vindemiatrix`). Outside the sandbox, set `KD_TTS_CMD` to bind to any local TTS.

## The engine, at a glance

```
script.yml ──▶ resolve_script ──┬─▶ synthesise_all ──▶ audio/*.mp3
                                │                          │
                                │           ┌──────────────┴──── build_audio_mix ──▶ audio_mix.m4a
                                │           │                                              │
                                ▼           ▼                                              ▼
                        build_beat_timeline (per-line start/end) ──▶ renderers (16x9,9x16,1x1) ──▶ video/*.mp4
                                                                              │
                                                                              └─▶ metadata.json + embed.html
```

Design tokens are constants in `kd/render_common.py`: cream paper `(245,236,216)`, ink `(28,24,22)`, ochre `(176,118,44)`, red-brown `(150,60,40)`; Noto Serif for the type. Active portraits scale 1.15× with a warm brightness bump; idle portraits desaturate to 0.75 and dim to 0.75.

## Distribution

| Platform  | Aspect | How it publishes                                                              |
|-----------|--------|--------------------------------------------------------------------------------|
| GitHub    | all    | `publish.py --target github` commits + pushes. Source of truth for everything. |
| Substack  | 1:1    | `--target substack` writes `embed.substack.html` pointing at the GitHub raw URL. Paste that into a Substack HTML embed. |
| YouTube   | 16:9   | `--target youtube` uploads via the YouTube Data connector.                     |
| TikTok    | 9:16   | Hand-post via app. `dist/captions/tiktok.txt` is your caption.                 |
| Instagram | 9:16 / 1:1 | Hand-post via Creator Studio. Reels = 9:16, feed = 1:1.                    |
| X         | 1:1    | Hand-post. `dist/captions/x.txt` is your post text.                            |

TikTok/IG/X don't have reliable programmatic upload paths for individuals, so the export pack is intentionally optimised for a "10-second copy-paste from your laptop" flow.
